# LOOP — Requirement (สรุปจาก Prototype)

> เอกสารนี้สรุปมาจากการอ่าน source ทั้งหมดใน `Prototype เว็บ C2C/` (`LOOP Home.dc.html`, `LOOP Home (standalone).html`, `LOOP Home-print-4k2gs0.dc.html`, `image-slot.js`, `support.js`, `.image-slots.state.json`) เป็น **functional requirement ล้วนๆ** (ไม่ผูกกับ tech stack ใดๆ) เพื่อใช้เป็น scope อ้างอิงตอนสร้างระบบจริง

---

## 1. ภาพรวมโปรเจกต์

**LOOP** ไม่ใช่เว็บซื้อขายของมือสองแบบ C2C ทั่วไป แต่เป็น **แพลตฟอร์มให้เช่าสินค้าระหว่างบุคคล (P2P rental marketplace)** — ผู้ใช้นำของที่มีอยู่มาปล่อยเช่าเป็นรายวัน แทนที่จะขายขาด

- Tagline: "Rent anything from people near you." / "เช่าอะไรก็ได้จากคนใกล้ตัวคุณ"
- ราคาสินค้าคิดเป็น **บาท/วัน** (`฿NNN /วัน`) ไม่ใช่ราคาขายครั้งเดียว
- ทุกการเช่ามี **เงินประกันความเสียหาย (refundable deposit)** และผ่านการ **verify ตัวตนผู้ใช้ (ID verify)**
- รองรับ 2 ภาษา: ไทย (default) / อังกฤษ, สกุลเงินบาท (฿), ปักหมุดกลุ่มเป้าหมายเป็นประเทศไทย (กรุงเทพฯ, เชียงใหม่, ภูเก็ต, พัทยา, ปาย) และใช้ LINE ID เป็นช่องทางติดต่อคู่กับเบอร์โทร
- ชื่อ "LOOP" สื่อถึงของที่หมุนเวียน (ให้เช่า–คืน–ให้เช่าใหม่) มากกว่าธุรกรรมซื้อขายทางเดียว
- ท้าย footer ระบุ "A concept prototype" — ของเดิมเป็น **prototype เชิงแนวคิด** ไม่ใช่ระบบ production เต็มรูปแบบ (auth, การชำระเงิน, การยืนยันตัวตนล้วนเป็นของจำลอง/demo)

## 2. User Roles (ตามพฤติกรรมการใช้งาน ไม่ใช่ table สิทธิ์จริง)

| Role | คำอธิบาย |
|---|---|
| **ผู้เช่า (Renter)** | ค้นหา/เช่าสินค้าคนอื่น, มีหน้า "My Rentals" ดูประวัติการเช่าของตัวเอง |
| **เจ้าของสินค้า (Lister/Owner)** | ลงประกาศสินค้าให้เช่า, มีหน้า "My Listings" จัดการประกาศ + อนุมัติ/ปฏิเสธคำขอเช่า |
| **ผู้ดูแลระบบ (Admin)** | เข้าหน้า Admin panel แยกต่างหาก จัดการ category, user, สินค้า, การเงิน, ตั้งค่าระบบ |

หมายเหตุ: ใน prototype **ผู้เช่ากับเจ้าของสินค้าคือ user คนเดียวกัน** (บัญชีเดียวมีทั้งสองความสามารถ) และการสลับไป admin เป็นปุ่ม toggle demo เท่านั้น ไม่มีระบบ auth/สิทธิ์จริงรองรับ — เป็นสิ่งที่ต้องออกแบบเพิ่มตอนสร้างระบบจริง

## 3. ขอบเขตฟีเจอร์ (Feature Scope)

### ✅ ฟีเจอร์ที่ prototype แสดงไว้ครบ (ใช้เป็น baseline ของ Phase แรก)

| หมวด | ฟีเจอร์ |
|---|---|
| หน้าแรก (Home) | Hero + search, สินค้ามาแรง (จำกัด 8 ชิ้น), หมวดหมู่ยอดนิยม (5 หมวด), วิธีใช้งาน 3 ขั้นตอน, รีวิวจากผู้ใช้, CTA ชวนลงประกาศ |
| ร้านค้า/เบราส์สินค้า (Shop) | ค้นหาด้วยคำ + filter หมวดหมู่ (chips), แสดงผลลัพธ์เต็ม, empty state เมื่อไม่พบ |
| รายละเอียดสินค้า (Product Detail) | gallery หลายรูป (ลูกศร/dot), รายละเอียดตามหมวดหมู่, รายการ "สินค้าคล้ายกัน" เทียบราคา, ปุ่ม "ขอเช่า" |
| ระบบจอง/เช่า (Checkout) | เลือกวันเริ่ม–สิ้นสุด, จุดนัดรับสินค้า, ข้อมูลติดต่อ (เบอร์โทร/LINE ID), หมายเหตุถึงเจ้าของ, สรุปราคา (ค่าเช่า × จำนวนคืน + ค่าธรรมเนียม + เงินประกัน), หน้ายืนยันสำเร็จ, ช่วงเวลายกเลิกคำขอได้ฟรี 10 นาทีหลังยืนยัน (มี countdown) |
| ลงประกาศสินค้า (List Item) | อัปโหลดรูป, ชื่อสินค้า, เลือกหมวดหมู่, ราคา/วัน, รายละเอียด, จุดรับสินค้า (เลือกได้หลายจุด + เพิ่มจุดใหม่), ที่อยู่สินค้า (แผนที่ตัวอย่าง + กรอกที่อยู่), หน้ายืนยันส่งประกาศสำเร็จ |
| สมัครสมาชิก + ยืนยันตัวตน (Sign Up) | เลือกประเภทบัญชี (บุคคล/ร้านค้า), กรอกข้อมูลพื้นฐาน, อัปโหลดรูปบัตรประชาชน + demo OCR ดึงข้อมูลจากรูป, ยืนยันใบหน้าผ่านมือถือ (QR scan, จำลอง), ยืนยัน OTP (อีเมล/เบอร์โทร), หน้าสำเร็จ |
| จัดการประกาศของฉัน (My Listings) | แท็บ "ประกาศของฉัน" (สถานะ active/รอตรวจสอบ/พักไว้, ปุ่มพัก/ลบ) และแท็บ "คำขอเช่า" (pending→approve/decline, approved→ยืนยันจัดส่งพร้อมรูปสภาพสินค้า, shipped→ยืนยันรับคืนพร้อมรูป) |
| ประวัติการเช่าของฉัน (My Rentals) | รายการเช่าตามสถานะ (รอยืนยัน/รอรับสินค้า/กำลังใช้งาน/ส่งคืนแล้ว/เสร็จสิ้น/ยกเลิก), ปุ่มยกเลิกคำขอในช่วง 10 นาทีแรก, ปุ่มคืนสินค้าพร้อมแนบรูป |
| Favorites/Saved | กดหัวใจบันทึกสินค้าที่สนใจ, badge นับจำนวนบน header |
| Admin: Dashboard | สรุปจำนวนสินค้า/ผู้ใช้ (+รอตรวจสอบ)/คำสั่งเช่า (+pending)/มูลค่าการเช่ารวม/จำนวนหมวดหมู่/ประกาศที่ active + สัดส่วนสินค้าต่อหมวดหมู่ |
| Admin: หมวดหมู่ (Category master) | CRUD เต็มรูปแบบ (เพิ่ม/แก้ไข/ลบ, นับจำนวนสินค้าในหมวด) |
| Admin: ผู้ใช้ (Users) | ตารางผู้ใช้ทั้งหมด, อนุมัติ/ปฏิเสธผู้ใช้ที่รอตรวจสอบ |
| Admin: สินค้า (Products) | ตารางสินค้าทั้งหมดในระบบ (read-only): รูป, ชื่อ, หมวดหมู่, ผู้ขาย, ราคา, คะแนน, ที่ตั้ง |
| Admin: การเงิน (Payments) | สรุปยอดเงินที่เคลียร์แล้ว/ค่าธรรมเนียมที่เก็บได้/ยอดค้างจ่าย + ตารางธุรกรรมพร้อมสถานะการชำระเงิน |
| Admin: ตั้งค่าระบบ (Settings) | ชื่อแพลตฟอร์ม, สกุลเงิน, % ค่าธรรมเนียมบริการ, อีเมลฝ่ายสนับสนุน, สวิตช์ "บังคับยืนยันตัวตนก่อนสมัคร", สวิตช์ "โหมดปิดปรับปรุงระบบ" |

### 🔜 สิ่งที่ยังไม่มีจริงใน prototype — ต้องออกแบบ/สร้างเพิ่มตอนทำระบบจริง

| หัวข้อ | รายละเอียด / เหตุผลที่ต้องเพิ่ม |
|---|---|
| ระบบ Auth/สิทธิ์จริง | ปัจจุบัน "สลับเป็น admin" เป็นปุ่ม demo ล้วนๆ ไม่มี login/JWT/role guard จริง |
| เชื่อมข้อมูลปลายทาง (Backend/DB) | ทุกอย่างเป็น in-memory state, สินค้า/ผู้ใช้/คำสั่งเช่าเป็นข้อมูล seed คงที่ ไม่มี API/DB จริง |
| เชื่อม flow "ยืนยันเช่า" และ "ส่งประกาศ" เข้ากับประวัติจริง | กดยืนยันเช่า/ส่งประกาศใน prototype แค่ตั้ง flag "สำเร็จ" แต่ไม่ได้เพิ่มรายการใหม่ใน "My Rentals"/"My Listings"/Admin Orders — ต้องเชื่อมให้ครบวงจรจริง |
| ผู้เช่า/เจ้าของสินค้าคนละบัญชี | ปัจจุบันข้อมูลผู้ขาย (shop name/avatar) สุ่มจาก id สินค้า ไม่ได้ผูกกับ user จริงที่เป็นเจ้าของ |
| ยืนยันตัวตน (KYC) จริง | OCR อ่านบัตรประชาชน และยืนยันใบหน้าผ่านมือถือ เป็นการจำลองล้วนๆ (ปุ่ม "(demo)"/"simulate scan complete") ต้องต่อ OCR/liveness-check จริง |
| ระบบชำระเงินจริง | ค่าธรรมเนียมบริการใน checkout hardcode 10% ไม่ได้อ่านจากค่า "% ค่าธรรมเนียม" ที่ตั้งใน Admin Settings เลย ต้องแก้ให้เป็นค่าเดียวกันและต่อ payment gateway จริง |
| แผนที่จริง | ช่องเลือกที่ตั้งสินค้าใน "ลงประกาศ" เป็นภาพ mock กริด ไม่ใช่แผนที่ใช้งานได้จริง |
| แกลเลอรีรูปต่อสินค้าจริง | รูปสินค้าในหน้ารายละเอียดใช้ชุดรูป Unsplash ตายตัวต่อหมวดหมู่ ไม่ใช่รูปที่อัปโหลดจริงต่อสินค้าแต่ละชิ้น |
| Help center / How it works | เป็นลิงก์ตายใน footer/เมนู ยังไม่มีหน้าเนื้อหาจริง |
| สวิตช์ "บังคับยืนยันตัวตน" / "โหมดปิดปรับปรุง" ใน Settings | เป็นสวิตช์เปล่า ไม่ได้ไปมีผลกับส่วนอื่นของระบบจริง ต้อง implement การบังคับใช้จริง |
| Rating/Review จริง | ดาวคะแนน + จำนวนรีวิวเป็นข้อมูล seed คงที่ ไม่มีระบบให้ผู้ใช้รีวิวจริงหลังเช่าเสร็จ |

## 4. รายละเอียดฟังก์ชันแต่ละหน้า (Functional Spec)

### 4.1 Header/เมนูหลัก (ทุกหน้า)
- โลโก้ LOOP (กดกลับหน้าแรก), แท็บ Home/Shop
- ปุ่ม "Admin panel" (แสดงเฉพาะ admin)
- ปุ่มสลับภาษา EN/TH
- ปุ่ม "ลงประกาศให้เช่า"
- ไอคอนหัวใจ (Saved) พร้อม badge จำนวน
- เมนู avatar บัญชี: My rentals, My listings, How it works, Help center, Sign up, สลับ admin/user

### 4.2 Home
- แถบค้นหา, การ์ดสินค้ามาแรง (badge "Popular"/"New", ปุ่มดูรายละเอียด, ปุ่มหัวใจ, ชื่อร้าน, คะแนน, ที่ตั้ง, ราคา/วัน) — จำกัด 8 ชิ้น
- Browse by category: 5 หมวด (กล้อง, อิเล็กทรอนิกส์, แฟชั่น, กิจกรรมกลางแจ้ง, เครื่องมือ/บ้าน) พร้อมจำนวนสินค้าต่อหมวด
- How it works: หา → จองวัน → รับ-คืนสินค้า
- รีวิวจากผู้ใช้ 3 รายการ
- CTA ชวนลงประกาศสินค้าที่ไม่ได้ใช้

### 4.3 Shop
- แถบค้นหา + filter หมวดหมู่ (chips: ทั้งหมด/อิเล็กทรอนิกส์/กล้อง/แฟชั่น/กิจกรรมกลางแจ้ง/เครื่องมือ)
- แสดงผลลัพธ์เต็ม (ไม่จำกัด 8), หัวข้อเปลี่ยนตาม query หรือหมวดหมู่ที่เลือก, empty state "ไม่พบสินค้า"

### 4.4 รายละเอียดสินค้า (modal overlay)
- Gallery รูป (ลูกศรซ้าย-ขวา + dot indicator)
- ชื่อร้าน/avatar, คะแนน+จำนวนรีวิว, ราคา/วัน, รายละเอียดสินค้าตามหมวดหมู่
- "สินค้าคล้ายกัน" สูงสุด 4 รายการในหมวดเดียวกัน เรียงตามราคา
- ปุ่ม "ขอเช่า" → เปิด Checkout

### 4.5 Checkout (ขอเช่า)
- เลือกวันเริ่ม/สิ้นสุด, จุดรับสินค้า (dropdown จากจุดที่เจ้าของเปิดไว้), เบอร์โทร, LINE ID, หมายเหตุ (optional)
- สรุปราคา: ค่าเช่า×จำนวนคืน, ค่าธรรมเนียมบริการ, เงินประกันคืนได้ (2 เท่าของค่าเช่า/วัน), ยอดที่ต้องจ่ายวันนี้
- ปุ่ม "ยืนยันคำขอเช่า" → หน้าสำเร็จ
- ยกเลิกคำขอได้ฟรีภายใน 10 นาทีหลังยืนยัน (มี countdown + dialog ยืนยันการยกเลิก)

### 4.6 ลงประกาศสินค้า
- อัปโหลดรูป 1 รูป, ชื่อสินค้า, หมวดหมู่ (dropdown), ราคา/วัน, รายละเอียด (textarea)
- จุดรับสินค้า: รายการ chip ที่เลือก/ลบได้ + เพิ่มจุดใหม่
- ที่ตั้งสินค้า: ภาพแผนที่ตัวอย่าง + ช่องกรอกที่อยู่แบบข้อความ
- ปุ่มส่งประกาศ → หน้า "ส่งประกาศสำเร็จ รอตรวจสอบ" + ปุ่มลงประกาศเพิ่ม

### 4.7 สมัครสมาชิก (Sign Up)
ขั้นตอน: **กรอกฟอร์ม → ยืนยัน OTP → สำเร็จ**
- เลือกประเภทบัญชี (บุคคล/ร้านค้า), Username, Email, เบอร์โทร, รหัสผ่าน
- อัปโหลดรูปบัตรประชาชน + ปุ่ม "ดึงข้อมูลจากรูป (demo)" แสดงผล OCR จำลอง (ชื่อ, เลขบัตร, วันเกิด, วันหมดอายุ)
- ยืนยันใบหน้าผ่านมือถือ: เปิด QR code ให้สแกน → รอผล → ปุ่มจำลอง "สแกนสำเร็จ (demo)"
- ยืนยัน OTP: เลือกช่องทาง (อีเมล/เบอร์โทร) → กรอกรหัส 6 หลัก → สร้างบัญชีสำเร็จ

### 4.8 จัดการประกาศของฉัน (My Listings)
- แท็บ "ประกาศของฉัน": รูป/ชื่อ/ราคา/สถานะ (active/รอตรวจสอบ/พักไว้), ปุ่มพัก-เริ่มใหม่, ปุ่มลบ
- แท็บ "คำขอเช่า" (Orders): รูป/ชื่อ/ผู้เช่า/วันที่/ยอดรวม/สถานะ
  - `pending` → ปุ่มอนุมัติ/ปฏิเสธ
  - `approved` → แสดงเบอร์โทร/LINE ผู้เช่า + ปุ่ม "ยืนยันจัดส่ง" (ต้องแนบรูปสภาพสินค้าก่อนส่ง)
  - `shipped` → ปุ่ม "ยืนยันรับคืนสินค้า" (ต้องแนบรูปสภาพตอนรับคืน)
- pipeline สถานะ: pending → approved → shipped → completed (หรือ declined)

### 4.9 ประวัติการเช่าของฉัน (My Rentals)
- รายการ: รูป/ชื่อ/วันที่/ยอดรวม/สถานะ (รอยืนยัน/รอรับสินค้า/กำลังใช้งาน/ส่งคืนแล้ว/เสร็จสิ้น/ยกเลิก)
- `pending` ภายใน 10 นาทีแรก → แสดง countdown + ปุ่มยกเลิกคำขอ
- `active` → ปุ่ม "คืนสินค้า" (ต้องแนบรูปสภาพตอนคืนก่อนส่งคำขอคืน)
- กดแถวเปิดดูรายละเอียดสินค้านั้น

### 4.10 Admin — Dashboard
- การ์ดสถิติ: จำนวนสินค้าทั้งหมด, จำนวนผู้ใช้ (+ จำนวนที่รอตรวจสอบ), จำนวนคำสั่งเช่า (+ pending), มูลค่าการเช่ารวม, จำนวนหมวดหมู่, จำนวนประกาศที่ active
- สัดส่วนสินค้าต่อหมวดหมู่ (bar/percentage)

### 4.11 Admin — หมวดหมู่ (Category master)
- ตาราง: ชื่อ / จำนวนสินค้า / actions — เพิ่ม (inline), แก้ไข (inline), ลบ

### 4.12 Admin — ผู้ใช้ (Users)
- ตาราง: ชื่อ, อีเมล, วันที่สมัคร, ประเภท (บุคคล/ร้านค้า), สถานะ (รอตรวจสอบ/อนุมัติ/ปฏิเสธ)
- ผู้ใช้ที่ pending → ปุ่มอนุมัติ/ปฏิเสธ

### 4.13 Admin — สินค้า (Products)
- ตาราง read-only: รูป, ชื่อสินค้า, หมวดหมู่, ผู้ขาย, ราคา, คะแนน, ที่ตั้ง + จำนวนสินค้ารวม

### 4.14 Admin — การเงิน (Payments)
- การ์ดสรุป: ยอดเคลียร์แล้ว, ค่าธรรมเนียมที่เก็บได้, ยอดค้างจ่าย
- ตารางธุรกรรม: สินค้า, ผู้เช่า, จำนวนเงิน, ค่าธรรมเนียม, สถานะการชำระ (จ่ายแล้ว/รอดำเนินการ/คืนเงินแล้ว)

### 4.15 Admin — ตั้งค่าระบบ (Settings)
- ชื่อแพลตฟอร์ม, สกุลเงิน, % ค่าธรรมเนียมบริการ, อีเมลฝ่ายสนับสนุน
- สวิตช์: บังคับยืนยันตัวตนก่อนสมัคร, โหมดปิดปรับปรุงระบบ

## 5. Data Model (functional — จาก field ที่พบใน prototype)

**Product (สินค้า)**
`id, title, หมวดหมู่, ราคา/วัน, คะแนน, จำนวนรีวิว, ที่ตั้ง, badge (Popular/New), รูปภาพ (gallery)`
> หมายเหตุ: ยังไม่มีความสัมพันธ์กับ "เจ้าของสินค้า" จริง — ต้องเพิ่ม field ผู้ขาย/เจ้าของที่ผูกกับบัญชี user

**Category (หมวดหมู่)**
`id, ชื่อ, จำนวนสินค้าในหมวด`

**User (ผู้ใช้)**
`id, ชื่อ, อีเมล, วันที่สมัคร, สถานะ (รอตรวจสอบ/อนุมัติ/ปฏิเสธ), ประเภทบัญชี (บุคคล/ร้านค้า)`

**Order / คำขอเช่า (มุมเจ้าของสินค้า)**
`id, สินค้า, ผู้เช่า, เบอร์โทร, LINE ID, วันเริ่ม-สิ้นสุด, ยอดรวม, สถานะ (pending/approved/declined/shipped/completed)`

**Listing (ประกาศของฉัน — มุมเจ้าของ)**
`id, สินค้าอ้างอิง, สถานะ (active/รอตรวจสอบ/พักไว้)`

**RentalHistory (ประวัติเช่า — มุมผู้เช่า)**
`id, สินค้าอ้างอิง, วันเริ่ม-สิ้นสุด, ยอดรวม, สถานะ (pending/confirmed/active/returning/completed/cancelled), เวลาที่ขอเช่า (ใช้คำนวณหน้าต่างยกเลิก 10 นาที)`

**PickupOption (จุดรับสินค้า)**
`id, ชื่อจุดรับ, เปิดใช้งานหรือไม่`

**Settings (ตั้งค่าระบบ)**
`ชื่อแพลตฟอร์ม, สกุลเงิน, % ค่าธรรมเนียมบริการ, อีเมลฝ่ายสนับสนุน, บังคับยืนยันตัวตน (bool), โหมดปิดปรับปรุง (bool)`

## 6. User Flow หลัก

1. **เช่าสินค้า**: ค้นหา/เบราส์ → ดูรายละเอียดสินค้า → ขอเช่า (เลือกวัน+จุดรับ+ข้อมูลติดต่อ) → ยืนยัน → (ยกเลิกได้ใน 10 นาที) → เจ้าของอนุมัติ → เจ้าของยืนยันจัดส่ง (แนบรูป) → ผู้เช่าใช้งาน → คืนสินค้า (แนบรูป) → เจ้าของยืนยันรับคืน (แนบรูป) → เสร็จสิ้น
2. **ลงประกาศให้เช่า**: กรอกฟอร์มสินค้า + รูป + ราคา/วัน + จุดรับ + ที่ตั้ง → ส่งประกาศ → รอตรวจสอบ → active
3. **สมัครสมาชิก + ยืนยันตัวตน**: กรอกข้อมูล → อัปโหลดบัตรประชาชน (+OCR) → ยืนยันใบหน้าผ่านมือถือ → ยืนยัน OTP → สร้างบัญชี
4. **แอดมินอนุมัติผู้ใช้/ตรวจสอบสินค้า**: ผู้ใช้/ประกาศใหม่เข้าสถานะ pending → แอดมินอนุมัติหรือปฏิเสธใน Admin panel

## 7. ข้อควรระวัง/สิ่งที่ prototype ยังไม่สมบูรณ์ (พบจากโค้ดโดยตรง)

- การกด "ยืนยันคำขอเช่า" และ "ส่งประกาศ" ใน prototype **ไม่ได้เพิ่มข้อมูลจริงเข้าประวัติ** (My Rentals / My Listings / Admin Orders) — เป็นเพียงข้อความสำเร็จ ต้องออกแบบให้ผูกกันจริงเมื่อสร้างระบบจริง
- ค่าธรรมเนียมบริการใน Checkout **hardcode 10%** ไม่ได้อ้างอิงค่าที่ตั้งใน Admin Settings — ต้องทำให้เป็นค่าเดียวกัน
- การยืนยันตัวตน (OCR บัตรประชาชน, ยืนยันใบหน้าผ่านมือถือ) เป็น **demo/จำลองทั้งหมด** ไม่ได้เชื่อมบริการจริง
- สวิตช์ "บังคับยืนยันตัวตน"/"โหมดปิดปรับปรุงระบบ" ใน Settings **ยังไม่มีผลกับส่วนอื่นของระบบ**
- ไม่มี backend/database จริง ข้อมูลทั้งหมดเป็นค่าคงที่ในหน่วยความจำฝั่ง client
- สลับบทบาท admin/user เป็นปุ่ม demo ไม่มีระบบ authentication/authorization จริงรองรับ
